1. Basic Description¶

The Data

The data are in 51 text files named XX.TXT, where XX is the postal code of one of the 50 US states or DC (for Washington D.C.). The data are encoded in the typical CSV format (comma separated values). Each line is a separate record, and consists of the following:

state (postal code), gender (M or F), year, namem, frequency

The frequency is the number of babies of the given gender born in the given year in the given state with the given name.

Names which occured less than five times are omitted for privacy reasons.

Each name is a single word.

Remark. If you skip to the end, you will see some plots of various things. The first of them (in blue) shows that the male/female ratio for the first few years of the data is way off.

df.name.apply(lambda x: len(x.split())).unique()

Remarks.

  1. According to the description of the data (included with the download), names with less than 5 occurences for a year of birth in any state are omitted (for privacy reasons). So this is not a complete record.

  2. Only names which are a single word are included. There are therefore many names (Mary Kate, Jean Paul, etc...) which are not correctly represented. I suspect that Mary Sue, Mary Kate, Mary Ashley, etc... are all recorded as Mary. This should skew Mary a lot.

The dataset contains 12,933 unique male names and 19,747 unique female names, for a total of 29,828 names (there is some overlap). Either male names are just more generic, or there is some recording bias in the data.

There are data for the years 1910 through 2013 (inclusive), although of course not every name occurred every year.

Remarks.

  1. We want to find the name with the largest increase/decrease in popularity since 1980. This is a little ambiguous, though. Do we just want to compare 2013 popularity with 1980 popularity? (This is the most direct--and simplest--interpretation of the question.) Or do we want to find the name that has had the greatest increase in popularity, starting in 1980? That is, if a name increase 50000% in popularity between 1980 and 1981, and then disappeared forever, it might be the one we want. Whereas it would definitely not be the answer to the first interpretation.

  2. Normally, one would measure increase in relative terms. But since there are probably quite a few names which appear in 2013 but not in 1980, and hence have a relative increase of infinity %, I will use a hybrid metric: the absolute change in relative popularity.. It is not very interesting to find a name which occurred three times in 2013 and not at all in 2013, and I certainly wouldn't want to claim that such a name has had the largest increase in popularity.

Here are a couple functions to find the largest increase / decrease in an array (ordered).

def max_inc(a):
    """
    Given an array a, returns the largest increase (ordered) 
    between elements of a (as well as the corresponding indices); 
    i.e. j, k, a[k] - a[j] where k > j
    """
    
    n = len(a)
    
    min_so_far = a[0]
    min_so_far_idx = 0
    max_diff = 0
    max_diff_idx = 0
    min_idx = 0
    
    for j in range(1,n):
        if a[j] - min_so_far > max_diff:
            max_diff = a[j] - min_so_far
            max_diff_idx = j
            min_idx = min_so_far_idx
        if a[j] < min_so_far:
            min_so_far = a[j]
            min_so_far_idx = j
            
    return min_idx, max_diff_idx, max_diff

def max_dec(a):
    """
    Given an array a, returns the largest decrease (ordered) 
    between elements of a (as well as the corresponding indices); 
    i.e. j, k, a[j] - a[k] where k > j
    """
    
    n = len(a)
    
    max_so_far = a[0]
    max_so_far_idx = 0
    max_diff = 0
    max_diff_idx = 0
    max_idx = 0
    
    for j in range(1,n):
        if max_so_far - a[j] > max_diff:
            max_diff = max_so_far - a[j]
            max_diff_idx = j
            max_idx = max_so_far_idx
        if a[j] > max_so_far:
            max_so_far = a[j]
            max_so_far_idx = j
            
    return max_idx, max_diff_idx, max_diff

Gender Ambiguous Names : Take 2¶

Let's look at how many gender neutral names there are (according to my less stringent relative metric) per year. This will be a bit more computationally intensive.

mgndf = mdf[mdf.index.isin(two_gender_names)]
fgndf = fdf[fdf.index.isin(two_gender_names)]

Let's compute the relative gender neutrality $r$ of the names as before:

$$ r_{\text{name}} := \frac{|\text{M occurences} - \text{F occurences}|}{\text{M occurences} + \text{F occurences}} $$

If a name had no occurences of either gender in a year, I'll get a NaN. That is good (since I won't be interested in those names for that year anyway).

I'll define a name to be gender neutral if

$$ r_{\text{name}} < 0.1. $$

Of course, this is totally arbitrary, but from the earlier statistics it looks reasonable.

g_ratio = ((mgndf.astype(float) - fgndf.astype(float)) / (mgndf + fgndf)).apply(abs)

ratios = (g_ratio < 0.1).sum() / (mdf.sum() + fdf.sum())

plt.figure(figsize=(13,7))

Unique Names (or maybe just Uncommon Names)¶

Since the data only include names with greater than 5 occurences, I can't actually search for the number of unique names per year. But I can look at how many uncommon names there are per year. As usual, I will look at relative popularity of the names.

rmdf = mdf / mdf.sum()
rfdf = fdf / fdf.sum()

Let's get a feel for what sort of frequencies we're dealing with here. Let's look at the relative frequencies of the least common names which do occur each year.

plt.figure(figsize=(13,7))
plt.plot(x, rmdf[rmdf>0].min(), label="Male");
plt.plot(x, rfdf[rfdf>0].min(), color=sns.color_palette()[2], label="Female");
plt.title('Smallest Relative Frequencies of Occuring Names per Year', fontsize=18);
plt.xlim(1910, 2014);
plt.legend(loc="upper right", fontsize=14);

Recall that the data itself look questionable for the first few years (the huge female/male ratio). So I'll choose 0.0004 as my cutoff for "uncommon" names (ignoring the big spike at the beginning).

Here are the raw numbers of "uncommon" names per year.

cutoff = 0.0002

mun = ((rmdf>0) & (rmdf<cutoff)).sum()
fun = ((rfdf>0) & (rfdf<cutoff)).sum()

plt.fill_between(x, 0, mun, color=sns.color_palette()[0], alpha=0.2);

plt.title('Number of Uncommon Names per Year', fontsize=18);
plt.legend(loc='upper left', fontsize=14);

As usual, the relative values are more interesting.

plt.figure(figsize=(13,7))
plt.plot(x, mun / mdf.sum(), label="Male");

plt.fill_between(x, 0, fun / fdf.sum(), color=sns.color_palette()[2], alpha=0.2);
plt.xlim(1910, 2014);
plt.title('Relative Number of Uncommon Names per Year', fontsize=18);
plt.legend(loc='upper left', fontsize=14);

We see a definite trend here. Names have been getting more varied since the 1930s.
